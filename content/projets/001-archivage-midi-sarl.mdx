**Archive Dossier** est un projet développé pour la société MIDI Sarl. Ce projet permet de gérer efficacement plusieurs aspects de la gestion d'archives, y compris les clients, leurs comptes, ainsi que la gestion des dossiers et des pièces jointes associées. Il propose aussi un système de gestion des documents et des métadonnées dynamiques.

## Fonctionnalités principales

- **Gestion des clients** : Créez, modifiez et gérez les informations des clients et leurs comptes.
- **Gestion des dossiers** : Chaque client peut avoir plusieurs dossiers, regroupant des informations spécifiques.
- **Pièces jointes** : Ajoutez et gérez les pièces jointes associées à chaque dossier.
- **Gestion documentaire** : Gérez les documents associés aux dossiers, y compris le type de document.
- **Métadonnées dynamiques** : Chaque type de document peut avoir des métadonnées spécifiques et dynamiques, permettant une flexibilité accrue dans la gestion des informations.

## Technologies utilisées

- [Next.js](https://nextjs.org/) : Framework React pour le développement d'applications web modernes.
- [TypeScript (TS)](https://www.typescriptlang.org) : Langage utilisé pour apporter une typage statique à JavaScript.
- [Tailwind CSS](https://tailwindcss.com/) : Framework CSS pour un design rapide et moderne.
- [Prisma](https://www.prisma.io/) : ORM pour une gestion des bases de données simple et efficace.
- [PostgreSQL](https://www.postgresql.org) : Base de données relationnelle utilisée pour stocker les informations.
- [Vercel](https://vercel.com/) : Plateforme de déploiement pour héberger le projet.
- [Git](https://git-scm.com) : Système de gestion de version pour le suivi du code.
- [GitHub](https://github.com/) : Plateforme de partage et de collaboration autour du code source.

## Objectifs du projet

- Automatiser et optimiser la gestion des archives clients.
- Faciliter l'accès rapide aux informations pertinentes pour chaque dossier.
- Offrir une gestion centralisée des documents et des métadonnées associées.
